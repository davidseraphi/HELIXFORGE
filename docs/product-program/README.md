# HelixForge category-defining product program

## Purpose

This program turns the HelixForge catalog from broad names into buildable
five-year product contracts. It is deliberately more ambitious than the
current code, but every ambition must resolve into capabilities, records,
authority rules, user journeys, proof, and delivery gates.

These documents are target-state specifications. They do not declare that the
existing prototypes are complete or safe.

`PROGRAM_MANIFEST.json` is the machine-readable index of every product sheet,
activation wave, shared contract, roadmap, and Kimi execution prompt.
`GLOSSARY.md` explains specialist terms in plain English.

## The category

HelixForge is a **sovereign work and discovery platform**. It gives a person or
organization one local-first foundation for creating software, knowledge,
workflows, decisions, markets, learning, finance, health research, physical
systems, and scientific work without giving one cloud or AI vendor permanent
control over the data or operating rules.

Its category-defining claim is not “many apps.” It is this:

> Every important act can be understood, authorized, observed while it runs,
> recovered when it fails, moved to another machine, and independently proved.

## Program laws

1. **Truth before theatre.** A capability is not live until a fresh gate proves
   it. Interfaces show partial, delayed, failed, and unknown states honestly.
2. **One foundation, real domains.** Products reuse HelixCore, but every product
   must earn its name with a real domain model and engine.
3. **Human authority.** Agents may prepare and execute narrow approved work.
   They may not silently expand their authority or approve themselves.
4. **Local-first and portable.** A useful path works on a user-owned machine.
   Export includes data, relationships, history, policies, and proof.
5. **Cross-platform by contract.** Windows, macOS, and Linux are release gates,
   not future notes.
6. **Recoverable by default.** Delete goes to a 30-day bin. Long work can pause,
   resume, cancel, retry safely, and show its real state.
7. **Proof travels with work.** Important results include sources, checks,
   approvals, identity, limits, and signed evidence where appropriate.
8. **Progressive power.** A new user sees a calm useful product. Advanced users
   can reveal deep controls without moving to a different tool.
9. **No vendor as a critical path.** Models, clouds, identity providers,
   custody providers, storage, and compute live behind replaceable adapters.
10. **High-stakes humility.** Legal, clinical, financial, biological, energy,
    space, and safety decisions stay within accountable human authority.

## Product boundaries

- **HelixCore** owns canonical logical identity, identity bindings, policy
  enforcement, provider-neutral capability contracts, grants, jobs, audit,
  storage, metering, deployment health, recovery primitives, and common UI
  states.
- **Aether integration** may attest a HelixCore project identity and act as the
  preferred external proof or capability-broker provider. It does not create a
  second canonical identity or policy model. HelixCore retains a local provider
  with the same contract, and neither provider exposes secret values to an agent.
- **Products** own domain truth and user workflows. They do not build private
  copies of identity, vault, audit, billing, or job control.
- **HelixAnvil** is a separate native portfolio project. It may host HelixCode
  experiences but is not catalog product 22.

## How to use this program

1. Repair the foundation before widening product code.
2. Choose one product and one delivery gate.
3. Create a feature packet containing only the capabilities needed for that
   gate.
4. Build vertical user journeys, not layers of unused infrastructure.
5. Run fresh release gates on Windows, macOS, and Linux.
6. Record actual source truth in the sheet before advancing the gate.

The master Kimi brief lives in `KIMI_MASTER_BUILD_PROMPT.md`. It must never be
used to start all products in parallel. It is an operating contract for staged
work over many sessions.
