# HelixCode Electron packaging & signing

## Dev

```powershell
pnpm --filter @helixforge/helix-code-web dev
pnpm --filter @helixforge/helix-code-web electron:dev
```

## Directory pack (unsigned)

```powershell
pnpm --filter @helixforge/helix-code-web desktop:pack
# output: projects/helix-code/web/dist-electron/
```

## Code signing (Windows) — org material under Desktop/.keys

**Never commit certs.** Org secrets live at:

`%USERPROFILE%\Desktop\.keys\helixforge\code-signing\`

### Bootstrap / pack (recommended)

```powershell
# Creates self-signed org cert if missing, exports PFX + password file, sets CSC_*
.\scripts\helix_code_org_codesign.ps1

# Same + electron-builder pack
.\scripts\helix_code_org_codesign.ps1 -Pack
```

Replace `helix-code-org.pfx` with your enterprise OV/EV cert when available; keep password only in `helix-code-org.password.txt` or CI secret store. See **`ENTERPRISE_CODESIGN.md`**.

**Local proof (self-signed):** `.\scripts\helix_code_org_codesign.ps1 -Pack` writes `projects/helix-code/web/dist-electron/win-unpacked/HelixCode.exe` signed with subject `CN=HelixForge Org Code Signing`. Windows may report `UnknownError` / untrusted for self-signed certs until the cert is imported into Trusted Publishers — that is expected for local org bootstrap.

### Icon

Place `projects/helix-code/web/build/icon.ico` (256×256 recommended). Generated by `scripts/helix_code_make_icon.ps1` if missing.

### Auto-update

1. Host releases (static feed: `latest.yml` + installers) on MinIO/S3/CDN.
2. Set `HELIX_CODE_UPDATE_URL` to the feed base (no trailing path to app).
3. Desktop shell loads `electron-updater` when the env is set (`electron/main.cjs`).

### Manual env

```powershell
$env:CSC_LINK="$env:USERPROFILE\Desktop\.keys\helixforge\code-signing\helix-code-org.pfx"
$env:CSC_KEY_PASSWORD=(Get-Content "$env:USERPROFILE\Desktop\.keys\helixforge\code-signing\helix-code-org.password.txt" -Raw).Trim()
pnpm --filter @helixforge/helix-code-web exec electron-builder --config electron-builder.yml --win dir
```

| Env | Purpose |
|-----|---------|
| `CSC_LINK` | Path or base64 of `.pfx` |
| `CSC_KEY_PASSWORD` | PFX password |
| `WIN_CSC_LINK` | Win-specific override |

macOS: `CSC_LINK` + Apple Developer ID; enable notarization via `APPLE_ID` / `APPLE_APP_SPECIFIC_PASSWORD` in CI.

## Auto-update

1. Host releases (S3/MinIO/GitHub Releases).
2. Set `HELIX_CODE_UPDATE_URL` to the feed base.
3. Wire `electron-updater` in `electron/main.cjs` when URL is set (optional product step).

## Architecture note

Electron loads the Next web UI (default `:3102`). Production packs may ship a sidecar `next start` or reverse-proxy to a local gateway; signing applies to the Electron binary/installer, not the Rust API.
